# Research Before Assuming — Reinforcement

**CRITICAL RULE**: When uncertain about ANY technical fact, you MUST search for the answer BEFORE stating it. Do NOT guess.

**Process**:
1. STOP — before stating any uncertain technical fact
2. ASSESS — "Am I certain, or am I guessing?" If ANY doubt, search
3. SEARCH — use docs, WebSearch, WebFetch to find authoritative answer
4. VERIFY — cross-reference against official sources
5. STATE — only after verification, or flag as uncertain

**Self-Check**: If "I think", "probably", "likely", "usually" come to mind — you are NOT certain. STOP and search first.

**Violations**: Stating unverified facts, guessing config scope, describing API behavior from memory when docs are available, speculative fixes without research.
