## No AI Attribution

**CRITICAL RULE**: NEVER include any AI attribution in code, commits, documentation, or any other output. All work MUST appear as standard human-authored content.

### Mandatory Process

1. **STOP** — Before finalizing ANY output (code, commit, docs, comments), scan it for AI attribution
2. **CHECK** — Search your output for: "AI", "generated", "co-authored", "assisted by", "powered by", "language model", "LLM", "GPT"
3. **REMOVE** — If ANY attribution is found, delete it entirely
4. **PROCEED** — Only after confirming zero attribution, submit the output

### Self-Check

If you are about to add a comment, footer, commit trailer, or metadata that mentions AI or any LLM — you are VIOLATING this rule. STOP and remove it NOW.

### When This Rule Applies

This rule is triggered when ANY of the following occur:
- Writing git commit messages
- Adding code comments or docstrings
- Creating or editing documentation files
- Generating any text output that will be saved to a file
- Creating pull requests, issues, or any VCS metadata
- Writing configuration files, READMEs, changelogs, or release notes

### Violations

The following are VIOLATIONS of this rule:
- Adding "Generated with [tool]" to any output
- Adding "Co-Authored-By" git trailers referencing AI tools
- Including comments like "// Generated by AI" or "# AI-assisted"
- Adding any metadata, footer, or attribution indicating AI involvement
- Including "Powered by [tool]" or similar phrases anywhere
- Adding AI attribution even when the user doesn't explicitly forbid it

### Example — Wrong Approach

```
git commit -m "Fix login bug

Co-Authored-By: AI <ai@example.com>"
```

```python
# Generated with AI - assisted development
def fix_login():
    pass
```

### Example — Correct Approach

```
git commit -m "Fix login bug"
```

```python
def fix_login():
    pass
```
